    **Create `scripts/run-sync.ps1`:**

    This is the PowerShell script that Windows Task Scheduler invokes. It resolves the
    repo root from the script's own location, sets the working directory, and runs the sync.

    ```powershell
    # scripts/run-sync.ps1
    # Weekly sync wrapper — invoked by Windows Task Scheduler.
    # Requires ANTHROPIC_API_KEY and GITHUB_TOKEN to be set as Windows user environment variables.

    $ErrorActionPreference = 'Stop'

    # Resolve repo root from this script's location (scripts/ -> parent = repo root)
    $repoRoot = Split-Path -Parent $PSScriptRoot
    Set-Location $repoRoot

    Write-Host "[sync] Starting dev.to sync at $(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')"
    Write-Host "[sync] Working directory: $repoRoot"

    # Verify required env vars are present before spending any API calls
    if (-not $env:ANTHROPIC_API_KEY) {
        Write-Error "[sync] ANTHROPIC_API_KEY is not set. Set it via: Start -> Edit environment variables for your account"
        exit 1
    }
    if (-not $env:GITHUB_TOKEN) {
        Write-Error "[sync] GITHUB_TOKEN is not set. Set it via: Start -> Edit environment variables for your account"
        exit 1
    }

    # Run the sync pipeline
    & pnpm sync:devto
    $exitCode = $LASTEXITCODE

    if ($exitCode -eq 0) {
        Write-Host "[sync] Completed successfully at $(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')"
    } else {
        Write-Error "[sync] Failed with exit code $exitCode"
        exit $exitCode
    }
    ```

    **Create `.github/workflows/sync-devto.yml`:**

    GH Actions fallback — workflow_dispatch only, no cron (scheduling is Windows Task Scheduler).
    Useful for: manual runs from GitHub UI, CI smoke tests, debugging without a local machine.

    ```yaml
    name: Sync dev.to articles (manual)

    # SCHEDULING ARCHITECTURE:
    # Primary trigger: Windows Task Scheduler on developer's PC
    #   Script: scripts/run-sync.ps1 (weekly, Monday 09:00 local time)
    #   Requires: ANTHROPIC_API_KEY + GITHUB_TOKEN as Windows user environment variables
    #
    # This workflow is a MANUAL FALLBACK only — run from GitHub Actions UI when needed.
    # No cron trigger here.
    on:
      workflow_dispatch:
        inputs:
          max_translations:
            description: 'Max articles to translate (default: 5)'
            required: false
            default: '5'
            type: string

    permissions:
      contents: write
      pull-requests: write
      issues: write

    concurrency:
      group: sync-devto
      cancel-in-progress: false

    jobs:
      sync:
        runs-on: ubuntu-latest
        env:
          MAX_TRANSLATIONS_PER_RUN: ${{ github.event.inputs.max_translations || '5' }}
        steps:
          - name: Checkout
            uses: actions/checkout@v4

          - name: Set up pnpm
            uses: pnpm/action-setup@v4
            with:
              version: 9.15.0

          - name: Set up Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '22'
              cache: 'pnpm'

          - name: Install dependencies
            run: pnpm install --frozen-lockfile

          - name: Run sync pipeline
            run: pnpm sync:devto
            env:
              ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
              GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
              MAX_TRANSLATIONS_PER_RUN: ${{ env.MAX_TRANSLATIONS_PER_RUN }}
    ```

    Create `scripts/setup-scheduled-task.ps1`:

    This is a **one-time setup script** the developer runs once (as regular user, not admin)
    to register the weekly sync task. It uses the current user's context so the task runs
    with access to the user's environment variables (ANTHROPIC_API_KEY, GITHUB_TOKEN).

    `-StartWhenAvailable` ensures the task runs on next PC start if the machine was off
    at the scheduled time (Monday 09:00) — prevents silent skips.

    ```powershell
    # scripts/setup-scheduled-task.ps1
    # Run ONCE to register the weekly dev.to sync task in Windows Task Scheduler.
    # Does not require admin — registers under current user account.
    #
    # Prerequisites before running this script:
    #   1. Set ANTHROPIC_API_KEY as a Windows user environment variable
    #   2. Set GITHUB_TOKEN as a Windows user environment variable (Fine-grained PAT)
    #   See docs/sync-pipeline.md for step-by-step instructions.
    #
    # To run:
    #   cd C:\Repo\blog_sertao
    #   powershell -ExecutionPolicy Bypass -File scripts/setup-scheduled-task.ps1
    #
    # To verify after running:
    #   Get-ScheduledTask -TaskName 'SertaoSeraCloud-Sync'
    #
    # To remove (if you want to unregister):
    #   Unregister-ScheduledTask -TaskName 'SertaoSeraCloud-Sync' -Confirm:$false

    $ErrorActionPreference = 'Stop'

    $taskName    = 'SertaoSeraCloud-Sync'
    $repoRoot    = Split-Path -Parent $PSScriptRoot          # C:\Repo\blog_sertao
    $wrapperPath = Join-Path $repoRoot 'scripts\run-sync.ps1'

    # Verify the wrapper exists before registering
    if (-not (Test-Path $wrapperPath)) {
        Write-Error "Wrapper not found at $wrapperPath. Run this script from the repo root."
        exit 1
    }

    # Build the action — runs PowerShell with the wrapper script
    $action = New-ScheduledTaskAction `
        -Execute 'powershell.exe' `
        -Argument "-NonInteractive -ExecutionPolicy Bypass -File `"$wrapperPath`"" `
        -WorkingDirectory $repoRoot

    # Weekly trigger — Monday at 09:00 local time
    # Change -DaysOfWeek or -At to adjust the schedule
    $trigger = New-ScheduledTaskTrigger `
        -Weekly `
        -DaysOfWeek Monday `
        -At '09:00AM'

    $settings = New-ScheduledTaskSettingsSet `
        -ExecutionTimeLimit  (New-TimeSpan -Hours 1) `
        -StartWhenAvailable              `  # run on next start if PC was off at trigger time
        -MultipleInstances IgnoreNew     `  # do not stack if a previous run is still going
        -RunOnlyIfNetworkAvailable            # need network for Forem API + GitHub API

    # Register under current user (no -RunLevel Highest needed — no admin ops in the script)
    Register-ScheduledTask `
        -TaskName  $taskName `
        -Action    $action `
        -Trigger   $trigger `
        -Settings  $settings `
        -Force                              # overwrite if task already exists

    Write-Host ""
    Write-Host "Task Scheduler entry registered:"
    Write-Host "  Name    : $taskName"
    Write-Host "  Script  : $wrapperPath"
    Write-Host "  Schedule: Weekly, Monday at 09:00 (local time)"
    Write-Host "  Catchup : StartWhenAvailable = true (runs on next PC start if missed)"
    Write-Host ""
    Write-Host "Verify with:"
    Write-Host "  Get-ScheduledTask -TaskName '$taskName'"
    Write-Host ""
    Write-Host "Trigger a manual test run with:"
    Write-Host "  Start-ScheduledTask -TaskName '$taskName'"
    Write-Host ""
    Write-Host "Before the first scheduled run, make sure these are set as Windows user env vars:"
    Write-Host "  ANTHROPIC_API_KEY  — from console.anthropic.com > API Keys"
    Write-Host "  GITHUB_TOKEN       — Fine-grained PAT (Contents+PRs+Issues write on sertaoseracloud/blog_sertao)"
    Write-Host "  See docs/sync-pipeline.md for full setup instructions."
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Windows Task Scheduler → run-sync.ps1 | Task runs as current user account — inherits user env vars (ANTHROPIC_API_KEY, GITHUB_TOKEN) |
| run-sync.ps1 → sync-devto.ts (Node.js) | PowerShell launches pnpm; env vars passed through process environment |
| sync-devto.ts → GitHub REST API | GITHUB_TOKEN PAT authenticates branch/PR/issue creation |
| sync-devto.ts → Anthropic API | ANTHROPIC_API_KEY authenticates Haiku translation calls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-02-17 | Information Disclosure | GITHUB_TOKEN in Windows env | mitigate | User env vars visible to current user only; not in source code; .gitignore covers .env files; PAT scope is minimal (3 permissions) |
| T-02-18 | Information Disclosure | ANTHROPIC_API_KEY in Windows env | mitigate | Same as above; Anthropic $5/month budget alert limits blast radius of key leak |
| T-02-19 | Denial of Service (cost) | Weekly Task Scheduler run | accept | MAX_TRANSLATIONS_PER_RUN=5 circuit breaker in script; weekly cadence limits runs to 52/year max |
| T-02-20 | Tampering | setup-scheduled-task.ps1 hardcoded path | mitigate | Script uses $PSScriptRoot to resolve paths dynamically — attacker cannot redirect execution by moving the repo |
| T-02-21 | Elevation of Privilege | Task Scheduler registration | accept | Task registers at user level (no -RunLevel Highest) — cannot perform admin operations; script does not modify system files |
| T-02-22 | Denial of Service | Multiple simultaneous Task runs | mitigate | MultipleInstances IgnoreNew prevents a second run from starting if one is already active |
</threat_model>
