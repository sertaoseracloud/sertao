    ```yaml
    name: Sync dev.to articles

    # SCHEDULING ARCHITECTURE (Option A):
    # This workflow uses workflow_dispatch ONLY — no cron here.
    # Daily scheduling (03:00 UTC / 00:00 BRT) is handled by a Claude Code /schedule routine
    # defined in .claude/schedules/sync-devto-schedule.md.
    # On-demand invocation is handled by a Claude Code RemoteTrigger
    # defined in .claude/triggers/sync-devto-trigger.md.
    # Both fire this workflow via GitHub REST API workflow_dispatch endpoint.
    # Manual runs are also possible from the GitHub Actions UI (workflow_dispatch input).
    on:
      workflow_dispatch:
        inputs:
          max_translations:
            description: 'Maximum articles to translate in this run (default: 5)'
            required: false
            default: '5'
            type: string

    permissions:
      contents: write        # create branches + commit markdown files
      pull-requests: write   # open draft PRs via GitHub REST API (PRBuilder)
      issues: write          # open GitHub Issues for canonical lint / glossary drift

    concurrency:
      group: sync-devto
      cancel-in-progress: false  # do NOT cancel in-progress syncs; let them finish

    jobs:
      sync:
        runs-on: ubuntu-latest
        env:
          MAX_TRANSLATIONS_PER_RUN: ${{ github.event.inputs.max_translations || '5' }}
        steps:
          - name: Checkout
            uses: actions/checkout@v4

          - name: Set up pnpm
            uses: pnpm/action-setup@v4
            with:
              version: 9.15.0

          - name: Set up Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '22'
              cache: 'pnpm'

          - name: Install dependencies
            run: pnpm install --frozen-lockfile

          - name: Run sync pipeline
            run: pnpm sync:devto
            env:
              ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
              GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
              MAX_TRANSLATIONS_PER_RUN: ${{ env.MAX_TRANSLATIONS_PER_RUN }}
    ```

    Implementation notes:
    - `concurrency.cancel-in-progress: false` — a sync in progress must not be killed mid-translation (would waste Anthropic API cost and leave partial file writes)
    - `workflow_dispatch.inputs.max_translations` allows the Claude Code scheduler and manual runs to override the circuit breaker for one-off runs (e.g., first-run catch-up)
    - `GITHUB_TOKEN` is automatically available in GH Actions runtime; the permission block above grants it pull-requests:write and issues:write
    - No `peter-evans/create-pull-request` step is needed because PRBuilder opens PRs directly via GitHub REST API (see Plan 02-03)
    - pnpm + Node setup is copied verbatim from `deploy.yml` (canonical pattern per PATTERNS.md)

    **Create `.claude/schedules/sync-devto-schedule.md`:**

    This file documents the Claude Code `/schedule` routine. A Claude Code agent running this schedule will call the GitHub REST API to fire `workflow_dispatch` on `sync-devto.yml` daily.

    ```markdown
    # Claude Code Schedule: sync-devto-daily

    ## Purpose
    Triggers the `sync-devto.yml` GitHub Actions workflow daily at 03:00 UTC (00:00 BRT).
    This replaces a GH Actions `schedule: cron:` trigger — Claude Code is the scheduler.

    ## Schedule
    - **Cron:** `0 3 * * *` (daily at 03:00 UTC = 00:00 BRT)
    - **Timezone:** UTC
    - **Human readable:** Every day at midnight (Brasília Time)

    ## What it does
    Fires a `workflow_dispatch` event on `.github/workflows/sync-devto.yml` via the
    GitHub REST API, which runs `pnpm sync:devto` in a GH Actions Ubuntu environment
    with `ANTHROPIC_API_KEY` and `GITHUB_TOKEN` available as secrets.

    ## Setup with Claude Code /schedule

    To activate this schedule, run the following in Claude Code:

    ```
    /schedule create
    name: sync-devto-daily
    cron: 0 3 * * *
    description: Daily dev.to sync pipeline trigger
    task: |
      Use the GitHub REST API to trigger the sync-devto workflow.
      POST to https://api.github.com/repos/sertaoseracloud/blog_sertao/actions/workflows/sync-devto.yml/dispatches
      with Authorization: Bearer {GITHUB_TOKEN} and body {"ref": "main"}.
      Log the response status. If the dispatch succeeds (204 No Content), log "sync-devto dispatched successfully".
      If it fails, log the error status and body for debugging.
    ```

    ## Required secrets for the Claude Code agent
    - `GITHUB_TOKEN` — a Fine-grained PAT or classic token with `workflow` scope for
      `sertaoseracloud/blog_sertao`. The agent uses this to call the dispatch endpoint.
      Store as a Claude Code environment variable or secret.
    - Note: The `ANTHROPIC_API_KEY` for the translation is a **GH Actions secret** —
      it is only needed inside the GH Actions job, not by the Claude Code agent itself.

    ## Equivalent GitHub Actions cron (for reference / fallback)
    If Claude Code scheduling is unavailable, add this trigger to `sync-devto.yml`:
    ```yaml
    schedule:
      - cron: '0 3 * * *'   # 03:00 UTC = 00:00 BRT
    ```
    This is NOT active by default (Option A architecture).

    ## Monitoring
    - GH Actions run history: https://github.com/sertaoseracloud/blog_sertao/actions/workflows/sync-devto.yml
    - Expected: one successful run per day; PRs appear in https://github.com/sertaoseracloud/blog_sertao/pulls
    ```

    # Claude Code RemoteTrigger: sync-devto-manual

    ## Purpose
    On-demand invocation of the `sync-devto.yml` workflow — for manual runs,
    webhook integration, or emergency re-syncs outside the daily schedule.

    ## Trigger type
    RemoteTrigger (Claude Code SDK)

    ## Setup with Claude Code RemoteTrigger

    To register this trigger in Claude Code:

    ```
    /trigger create
    name: sync-devto-manual
    description: On-demand trigger for the dev.to sync pipeline
    action: |
      Trigger the sync-devto GitHub Actions workflow via workflow_dispatch.
      POST to https://api.github.com/repos/sertaoseracloud/blog_sertao/actions/workflows/sync-devto.yml/dispatches
      with Authorization: Bearer {GITHUB_TOKEN} and body {"ref": "main"}.
      Optionally accept a max_translations parameter (default: 5) and include it in the
      dispatch inputs: {"ref": "main", "inputs": {"max_translations": "{max_translations}"}}.
      Return the dispatch response status to the caller.
    ```

    ## Manual invocation (Claude Code CLI)
    ```bash
    # Trigger with default max_translations=5
    claude trigger sync-devto-manual

    # Trigger with custom limit (e.g., catch-up run)
    claude trigger sync-devto-manual --max_translations=10
    ```

    ## Webhook integration
    External services (e.g., a CI system, a dev.to webhook proxy) can POST to the
    Claude Code RemoteTrigger endpoint to invoke an on-demand sync:

    ```
    POST {CLAUDE_CODE_REMOTE_TRIGGER_URL}/sync-devto-manual
    Content-Type: application/json
    Authorization: Bearer {CLAUDE_CODE_TRIGGER_SECRET}
    Body: {"max_translations": 5}
    ```

    ## Required secrets
    - `GITHUB_TOKEN` — same Fine-grained PAT used by the schedule (workflow scope for
      `sertaoseracloud/blog_sertao`).

    ## Relationship to daily schedule
    - `.claude/schedules/sync-devto-schedule.md` — fires automatically at 03:00 UTC
    - This trigger — fires on demand; also useful for testing after changes to the sync script

    ## Monitoring
    GH Actions run history: https://github.com/sertaoseracloud/blog_sertao/actions/workflows/sync-devto.yml
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Code agent → GitHub REST API | Agent calls workflow_dispatch endpoint using GITHUB_TOKEN with workflow scope |
| GH Actions → sync script | GH Actions injects ANTHROPIC_API_KEY and GITHUB_TOKEN as masked env vars |
| External webhook → Claude Code RemoteTrigger | External POST to trigger endpoint; must be authenticated with CLAUDE_CODE_TRIGGER_SECRET |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-17 | Spoofing | RemoteTrigger endpoint | mitigate | RemoteTrigger requires CLAUDE_CODE_TRIGGER_SECRET authentication; document requirement in trigger spec |
| T-02-18 | Elevation of Privilege | sync-devto.yml permissions | mitigate | Minimal permissions: contents:write + pull-requests:write + issues:write only; no id-token:write, no pages:write |
| T-02-19 | Information Disclosure | ANTHROPIC_API_KEY | mitigate | Stored as GH Actions masked secret — never in workflow YAML; never printed in logs; GH Actions masks any accidental echo |
| T-02-20 | Denial of Service (cost) | Claude Code schedule | mitigate | Schedule fires once daily; MAX_TRANSLATIONS_PER_RUN=5 circuit breaker in the script limits API spend; Anthropic $5/month budget alert is additional gate |
| T-02-21 | Tampering | workflow_dispatch inputs | accept | max_translations input is parsed as Number() in sync-devto.ts; non-numeric defaults to 5; no code injection path via this string field |
| T-02-22 | Denial of Service | concurrency.cancel-in-progress | mitigate | Set to false — prevents a second schedule trigger from aborting a running sync mid-translation; at most one sync run active at a time |
</threat_model>
