```astro
interface Props {

    </svg>
    Copiar link
  </button>
</nav>

<script is:inline>
  (function () {
    // Use querySelectorAll — PostLayout renders TWO ShareBar instances (top+bottom).
    // getElementById would only find the first; class-based selection handles both.
    var btns = document.querySelectorAll('.share-copy-btn');
    btns.forEach(function (btn) {
      btn.addEventListener('click', function () {
        navigator.clipboard.writeText(window.location.href).then(function () {
          btn.textContent = 'Copiado! ✓';
          btn.setAttribute('aria-label', 'Link copiado');
          btn.classList.add('share-btn--copied');
          setTimeout(function () {
            btn.textContent = 'Copiar link';
            btn.setAttribute('aria-label', 'Copiar link do artigo');
            btn.classList.remove('share-btn--copied');
          }, 2000);
        }).catch(function () {
          // Silent failure — clipboard API unavailable; button does nothing
        });
      });
    });
  })();
</script>
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Astro props → share URL construction | `title` and `url` from PostLayout props (from content schema) flow into share URLs via encodeURIComponent |
| Copy-link script → clipboard API | `window.location.href` written to clipboard; not rendered into DOM |
| User → social platforms | Share links open X/LinkedIn/WhatsApp with post title and URL; those platforms display their own previews |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-04 | Tampering (open redirect) | ShareBar share URL construction | mitigate | Share URLs use fixed template strings with `encodeURIComponent` applied to both title and url props. No user-controlled redirect target. Post title and canonical URL are Zod-validated build-time constants. |
| T-08-05 | Tampering (XSS) | copy-link is:inline script | accept | `navigator.clipboard.writeText(window.location.href)` is a write-only browser API — value is not rendered into DOM. No innerHTML or DOM insertion occurs. Silent catch prevents error message injection. |
| T-08-06 | Tampering | SVG inline icons | accept | SVGs are static constants hardcoded in the component template — no dynamic values, no external fetch, no script injection surface. |
</threat_model>
