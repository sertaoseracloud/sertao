## Spacing Scale

Declared values (all are multiples of 4; map to `--space-*` CSS custom properties from `global.css`):

| Token | CSS Var | Value | Usage |
|-------|---------|-------|-------|
| xs | `--space-1` | 4px | Icon-to-text gap within share buttons, tag hash gap |
| sm | `--space-2` | 8px | Gap between share buttons, consent checkbox gap |
| md | `--space-4` | 16px | Internal button padding, spacing between ShareBar and article heading |
| lg | `--space-5` | 24px | ShareBar horizontal padding within wrapper; /sobre social links gap |
| xl | `--space-6` | 32px | PostLayout article header margin-bottom (existing); /sobre photo margin |
| 2xl | `--space-7` | 48px | ShareBar vertical clearance from article (bottom instance); /sobre section gaps |
| 3xl | `--space-8` | 64px | OG endpoint is non-visual; used as section break in /sobre between bio and thesis |

Exceptions:
- Share button touch target minimum height: 44px (WCAG 2.5.3) — matches newsletter-submit-btn pattern.
- OG image canvas is 1200×630px — not on the 8-point scale; this is a fixed media format, not a UI layout unit.

Source: `src/styles/global.css` `:root` spacing vars + WCAG 2.5.3 minimum touch target from Phase 7 newsletter pattern.

## Color

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--abismo-profundo` | `#0A0F1E` | Page background, OG image background, ShareBar container background |
| Secondary (30%) | `--sub-nivel` | `#1B293C` | ShareBar button background (rest state), /sobre photo border/container |
| Accent (10%) | `--nucleo-eletrico` | `#00FFFF` | Reserved — see list below |
| Muted text | `--texto-secundario` | `#D1D9E6` | ShareBar label text (rest state), /sobre social link labels (rest), OG tag badge text |
| Hairline border | `--hairline` | `rgba(209, 217, 230, 0.14)` | ShareBar button border (rest), /sobre social link border (rest) |
| Hairline strong | `--hairline-strong` | `rgba(0, 255, 255, 0.32)` | ShareBar button border (hover/focus), /sobre social link border (hover) |

Destructive: none applicable in this phase (no delete/revoke actions in ShareBar, OG generation, or /sobre).

Accent (`#00FFFF`) reserved for exactly these elements in Phase 8:
1. OG image separator line (2px horizontal rule between title and author row)
2. OG image site wordmark "O Sertão será Cloud" text
3. ShareBar button hover state — border-color transitions to `--hairline-strong` and label color to `--nucleo-eletrico`
4. ShareBar copy-link button "Copiado! ✓" feedback label color
5. Focus ring on all ShareBar interactive elements (`:focus-visible` — inherits global rule)
6. /sobre social links hover color

Accent is NOT used for ShareBar button backgrounds, body text, or container fills.

Source: `src/styles/global.css` color tokens + CLAUDE.md WCAG rules + 08-CONTEXT.md D-04, D-06.

Light theme note: `[data-theme="light"]` overrides `--nucleo-eletrico` to `#284068` (cobalt). ShareBar and /sobre social links automatically inherit the correct value. OG image is dark-only (generated server-side, not theme-aware) — always uses `#00FFFF` hardcoded.

## Component Specifications

### ShareBar (`src/components/ShareBar.astro`)

Props: `title: string`, `url: string` (full canonical URL — PostLayout constructs from `SITE_URL + /posts/ + slug`).

Layout: single horizontal row, `display:flex; gap:var(--space-2); flex-wrap:wrap;` — buttons stack on narrow viewports.

Button anatomy (shared base):
```
[icon 16×16 SVG] [gap 4px] [text label]
height: 44px
padding: 0 var(--space-4) (0 16px)
border: 1px solid var(--hairline)
background: var(--sub-nivel)
font-family: 'Chakra Petch', sans-serif
font-size: 13px
font-weight: 600
letter-spacing: 0.12em
text-transform: uppercase
color: var(--texto-secundario)
transition: border-color 0.2s, color 0.2s
```

Hover/focus state:
```
border-color: var(--hairline-strong)
color: var(--nucleo-eletrico)
```

Four buttons (in order):

| Button | Element | Label | Icon | href / behavior |
|--------|---------|-------|------|-----------------|
| X | `<a>` | X | X bird SVG (16×16) | `https://twitter.com/intent/tweet?text={title}&url={url}` — `target="_blank" rel="noopener noreferrer"` |
| LinkedIn | `<a>` | LinkedIn | LinkedIn box SVG (16×16) | `https://www.linkedin.com/sharing/share-offsite/?url={url}` — `target="_blank" rel="noopener noreferrer"` |
| WhatsApp | `<a>` | WhatsApp | WhatsApp bubble SVG (16×16) | `https://wa.me/?text={title}%20{url}` — `target="_blank" rel="noopener noreferrer"` |
| Copiar link | `<button>` | Copiar link | Link/chain SVG (16×16) | `navigator.clipboard.writeText(window.location.href)` via `is:inline` script |

Copy-link feedback states:
- Rest: label "Copiar link", icon = link chain SVG
- Success (2 seconds): label "Copiado! ✓", color = `var(--nucleo-eletrico)` — then reverts
- No error state displayed (clipboard failure silently ignored — same pattern as CopyCode.astro; if clipboard API unavailable, button does nothing)

PostLayout insertion order (from 08-CONTEXT.md D-08):
```
<ShareBar title={title} url={canonicalUrl} />   ← top (before article header)
<article class="prose">
  ...
</article>
<CopyCode />
<ShareBar title={title} url={canonicalUrl} />   ← bottom (after article, before NewsletterEmbed)
<NewsletterEmbed />
<CommentsEmbed />
```

CSS class: `.share-bar` wrapper — add to Phase 8 section in `global.css`.

### OG Image Endpoint (`src/pages/og/[...slug].png.ts`)

Not a visible UI component — generates 1200×630 PNG at build time.

Satori flexbox layout (top-level column, full canvas):
```
Container: 1200×630, background #0A0F1E, padding 72px 80px, flex column, justify-content space-between

Top section (flex column, gap 24px):
  - Tag badge (if tags[0] exists): text "#" + tags[0].toUpperCase(), font Chakra Petch 22px, color #D1D9E6,
    border 1px solid rgba(209,217,230,0.3), padding 4px 12px, align-self flex-start
  - Title text: Space Grotesk 64px weight 600, color #FFFFFF, max 2 lines, line-height 1.15

Separator: width 100%, height 2px, background #00FFFF, margin 32px 0

Bottom row (flex row, align-items center, justify-content space-between):
  - Left: flex column, gap 4px
      - Author: "Cláudio Rapôso" Space Grotesk 28px weight 400, color #FFFFFF
      - Wordmark: "O Sertão será Cloud" Chakra Petch 24px weight 400, color #00FFFF
  - Right: site URL "sertaoseracloud.com" JetBrains Mono 18px, color rgba(209,217,230,0.6)
```

Title truncation: if title > 70 chars, render as-is (satori wraps at 2 lines automatically; add `textOverflow:'ellipsis'` on the title div for safety).

Tag fallback: if no tags, omit the badge div entirely — separator and bottom row layout unchanged.

Font loading: read WOFF2 files as ArrayBuffer from `public/fonts/` at build time using `fs.readFileSync`. Space Grotesk weight 600 (`space-grotesk-600.woff2`) and Chakra Petch weight 400 (`chakra-petch-400.woff2`) are the two required fonts. JetBrains Mono 400 (`jetbrains-mono-400.woff2`) for the URL string.

Output: `sharp` converts satori SVG to PNG buffer. `Content-Type: image/png`.

`og:image` URL constructed in PostLayout: `${SITE_URL}/og/${slug}.png` — passed as `image` prop to BaseLayout, which forwards to SEO.astro (existing `image` prop path, no changes needed to SEO.astro).

### /sobre Page (`src/pages/sobre.astro`)

Route: `/sobre`
Pattern: identical to `privacidade.astro` — `BaseLayout` + `Header` + `Footer` + `<article class="prose">`.

Layout structure (top-to-bottom within `.prose`):
```
1. Page header section
   h1: "Sobre" — Space Grotesk clamp(32px, 2.8vw, 44px), weight 600
   
2. Bio intro block (flex row on desktop, column on mobile)
   - Photo: <img src="/images/author.jpg" alt="Cláudio Rapôso"> — 160×160px, object-fit:cover,
     float:right on desktop (margin-left:32px, margin-bottom:16px), block centered on mobile
   - Bio paragraphs: .prose body (18px, 1.72 lh, --prose-fg)
   - Graceful degradation: if /images/author.jpg absent at build time, img element still renders
     (browser shows broken-image placeholder; layout does not collapse)

3. Thesis section
   h2: "Por que 'O Sertão será Cloud'?" — Space Grotesk 26px, weight 600, line-height 1.15
   - Essay paragraphs: .prose body (18px, 1.72 lh)

4. Social links row
   - Display: flex row, gap var(--space-5) (24px), flex-wrap:wrap
   - Four links: GitHub, LinkedIn, X, email (mailto)
   - Each: native <a>, Chakra Petch 13px weight 600, letter-spacing 0.14em, text-transform:uppercase,
     color: var(--texto-secundario), text-decoration:none
   - Hover: color var(--nucleo-eletrico)
   - External links: target="_blank" rel="noopener noreferrer" (GitHub, LinkedIn, X)
   - Email: mailto:engcfraposo@gmail.com (no target)
   - LinkedIn URL: add `linkedin: 'https://www.linkedin.com/in/cfraposo/'` to SOCIAL in consts.ts
     (D-16 — currently missing from consts.ts)
```

SEO: `title="Sobre — Cláudio Rapôso"`, `description` = 1–2 sentence bio summary (executor to draft matching AUTHOR.name from consts.ts). No `type="article"` — default `type="website"`.

## Interaction States Summary

| Component | State | Visual Change |
|-----------|-------|---------------|
| ShareBar `<a>` buttons | hover | `border-color → --hairline-strong`, `color → --nucleo-eletrico` |
| ShareBar `<a>` buttons | focus-visible | 2px `--nucleo-eletrico` outline, offset 3px (global rule) |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
