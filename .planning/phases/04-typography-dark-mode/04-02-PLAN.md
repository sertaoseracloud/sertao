```astro
<script>
  const btn = document.getElementById('theme-toggle') as HTMLButtonElement;
  const iconSun = document.getElementById('icon-sun') as SVGElement;
  const iconMoon = document.getElementById('icon-moon') as SVGElement;
  const html = document.documentElement;

  function getTheme(): string {
    return html.getAttribute('data-theme') || 'dark';
  }

  function applyTheme(theme: string): void {
    html.setAttribute('data-theme', theme);
    localStorage.setItem('theme', theme);
    if (theme === 'light') {
      iconSun.style.display = 'none';
      iconMoon.style.display = 'block';
      btn.setAttribute('aria-label', 'Mudar para modo escuro');
    } else {
      iconSun.style.display = 'block';
      iconMoon.style.display = 'none';
      btn.setAttribute('aria-label', 'Mudar para modo claro');
    }
  }

  // Sync icon state on mount — SSG HTML always renders dark-default;
  // this corrects the icon when the user has a stored or system light preference.
  applyTheme(getTheme());

  btn.addEventListener('click', () => {
    applyTheme(getTheme() === 'dark' ? 'light' : 'dark');
  });
</script>
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ThemeToggle button click → localStorage + data-theme | User input (click) writes to localStorage and sets HTML attribute |
| astro.config.mjs shikiConfig → Markdown files | Build-time only; no user input in production |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | ThemeToggle: localStorage 'theme' value | mitigate | `applyTheme()` calls `setAttribute('data-theme', theme)` only. Stored value only controls `[data-theme="light"]` CSS selector — no eval, no innerHTML, no navigation. Attacker can change CSS theme only, no security impact |
| T-04-05 | Information disclosure | Shiki token highlighting | accept | Shiki processes markdown at build time — no user input processed; output is static HTML with CSS vars |
| T-04-06 | Tampering | ThemeToggle icon-sun/icon-moon display state | accept | State only controls icon visibility via `style.display` — no security boundary crossed; worst case wrong icon shown |
</threat_model>
