<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dev.to API → pipeline | Forem API response is external — body_markdown content trusted for translation, not executed |
| GitHub Actions → repo | Sync pipeline writes files via GitHub REST API; GITHUB_TOKEN scoped to contents+PRs+issues write |
| Author → PR merge | Author is sole reviewer and approver — no CI code review gate; intentional for v1 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | glossary.json | mitigate | Glossary commits are audited in PR diff; changes require explicit author approval before merge |
| T-05-02 | Information Disclosure | ANTHROPIC_API_KEY | accept | Key is a GitHub Actions secret; never logged; standard GH Actions secret handling is sufficient for v1 |
| T-05-03 | Spoofing | canonical_url | mitigate | Author verifies canonical_url in PR diff before merge; SEO.astro renders it verbatim — malformed URL would be visible in view-source check (Task 3 checklist item) |
| T-05-04 | Repudiation | PR merge | accept | GitHub audit log records all merges with actor identity; sufficient for solo developer |
</threat_model>
