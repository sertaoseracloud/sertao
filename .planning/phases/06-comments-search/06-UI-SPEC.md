- Touch targets: all interactive icon buttons (search icon, ThemeToggle) maintain minimum 40×40px hit area — inherited from ThemeToggle pattern.

| Role | Token | Value (dark) | Usage |
|------|-------|--------------|-------|

**Accent (`--nucleo-eletrico`) reserved for in Phase 6:**

1. Tag chip `#` hash prefix (at 0.7 opacity — existing `.tag .hash` rule)
2. Tag chip hover state text (full opacity — existing `.tag:hover` rule)
3. Search icon button color (`color: var(--nucleo-eletrico)` — matches ThemeToggle icon pattern)
4. Pagefind `--pagefind-ui-primary` variable (result match highlights, focus states inside modal)
5. Focus ring on search modal close button and search input (`:focus-visible` global rule applies)

**NEVER use `--nucleo-eletrico` or `--chama-primaria` as body paragraph text** — `--texto-principal` is the only WCAG-safe body text color on dark background.

**WCAG compliance for Phase 6 new elements:**

- Tag chip default text: `--texto-secundario` (`#D1D9E6`) on `--sub-nivel` (`#1B293C`) = approximately 6.2:1 (AA)
- Tag chip hover text: `--nucleo-eletrico` (`#00FFFF`) on `--sub-nivel` (`#1B293C`) = approximately 9.5:1 (AAA)
- Search modal background: `--sub-nivel` (`#1B293C`) — adequate contrast for text within it
- Search input text: `--texto-principal` (`#FFFFFF`) on `--sub-nivel` = 14:1 (AAA)
- Modal overlay backdrop: `rgba(10, 15, 30, 0.88)` — dark enough to obscure page content without full black

## Component Specifications

### 1. Tag Chips (`.tag`) — extending existing styles

Tag chips already have CSS rules in `global.css` (section 08). Phase 6 converts them from `<span>` to `<a>` elements with navigation targets.

**Upgrade from `<span>` to `<a>`:**

- Change: `<span class="tag">` → `<a href="/tags/{tag}" class="tag">`
- Existing CSS transitions and hover states apply automatically to `<a>` elements — no CSS change needed.
- Tag chips on the homepage (`src/pages/index.astro`) currently use `<span>` — Phase 6 changes them to `<a>` links.

**Exact chip HTML structure:**
```html
<a href="/tags/{normalizedTag}" class="tag">
  <span class="hash" aria-hidden="true">#</span>{normalizedTag}
</a>
```

**Tag normalization rule:** lowercase + deduplicated. Applied in `getStaticPaths` and at render time. Example: `["AWS", "aws", "Cloud"]` → `["aws", "cloud"]`.

**Tag row on post cards:**
```html
<div class="tag-row">
  {tags.slice(0, 3).map(tag => (
    <a href={`/tags/${tag}`} class="tag">
      <span class="hash" aria-hidden="true">#</span>{tag}
    </a>
  ))}
</div>
```

Maximum 3 tags shown per post card. Tags truncated silently (no "+N more" indicator).

**Tag chip in PostLayout header (below post title):**
- Same `.tag` + `.tag-row` markup as cards.
- Placement: below `<h1>` in PostLayout article header, above the date line.
- Show all tags (no truncation on the post page itself).

**Hover state (already declared in global.css):**
```css
.tag:hover {
  border-color: var(--hairline-strong);
  color: var(--nucleo-eletrico);
}
```

No additional hover CSS needed — existing rule covers `<a>` elements.

```html

     style="display:none; position:fixed; inset:0; z-index:9000;
```

**Modal behavior (JavaScript in `<script>` block):**

- `Ctrl+K` anywhere on the page: open modal. Prevents default browser behavior.
- Click on backdrop (outside panel): close modal.
- `Escape` key: close modal.
- When opened: set `aria-label` on button to "Fechar busca"; move focus to Pagefind search input.
- When closed: set `aria-label` back to "Abrir busca"; return focus to search button.
- Modal `display`: toggle between `none` (closed) and `block` (open).

**Pagefind UI initialization (inside `<script>`):**
```typescript

    element: '#pagefind-search',
    showSubResults: false,
    translations: {
      placeholder: 'Buscar posts...',
      zero_results: 'Nenhum resultado para "[SEARCH_TERM]"',
    },
  });
});
```

Pagefind UI is initialized once on first modal open (lazy), not at page load.

```css

  --pagefind-ui-text: var(--texto-principal);            /* #FFFFFF — result text */
```

### 6. ThemeToggle Extension — Giscus Theme Bridge

**File to modify:** `src/components/ThemeToggle.astro`

**Extend `applyTheme(theme: string)` function** to also post a message to the Giscus iframe:

```typescript
function applyTheme(theme: string): void {
  html.setAttribute('data-theme', theme);
  localStorage.setItem('theme', theme);
  if (theme === 'light') {
    iconSun.style.display = 'none';
    iconMoon.style.display = 'block';
    btn.setAttribute('aria-label', 'Mudar para modo escuro');
  } else {
    iconSun.style.display = 'block';
    iconMoon.style.display = 'none';
    btn.setAttribute('aria-label', 'Mudar para modo claro');
  }
  // Giscus theme bridge — D-04
  const giscusFrame = document.querySelector<HTMLIFrameElement>('iframe.giscus-frame');
  if (giscusFrame?.contentWindow) {
    giscusFrame.contentWindow.postMessage(
      { giscus: { setConfig: { theme: theme === 'light' ? 'light' : 'dark' } } },
      'https://giscus.app'
    );
  }
}
```

This change is backward-compatible — `document.querySelector('iframe.giscus-frame')` returns `null` on pages without Giscus, so the `?.contentWindow` optional chain short-circuits safely.
