## Design System Palette Reality Check

The early planning docs (CONTEXT.md, ROADMAP) reference a light-first alias set (`--color-text-primary: #284068`, `--color-focus-ring: #284068`). These aliases are compatibility shims only — declared in `global.css` section "03b · ROADMAP Compatibility Aliases" with explicit note: "DO NOT use these aliases in components — use the canonical names."

The actual implemented design system is **dark-first**:

| Canonical Token | Value | Role |
|----------------|-------|------|
| `--abismo-profundo` | `#0A0F1E` | Page background (dominant surface) |
| `--sub-nivel` | `#1B293C` | Cards, elevated surfaces |
| `--texto-principal` | `#FFFFFF` | Body text, headings |
| `--texto-secundario` | `#D1D9E6` | Metadata, timestamps, muted text |
| `--nucleo-eletrico` | `#00FFFF` | Accent — links, icons, borders, highlights |
| `--chama-primaria` | `#1A3AC8` | Decorative background fills, logo mark |
| `--hairline` | `rgba(209, 217, 230, 0.14)` | Subtle borders |
| `--hairline-strong` | `rgba(0, 255, 255, 0.32)` | Emphasized borders |

**Focus ring correction:** `#284068` (dark navy) on `#0A0F1E` background yields near-zero contrast (~1.5:1) — NOT accessible. Focus ring MUST use `--nucleo-eletrico: #00FFFF` on this dark background. Contrast of `#00FFFF` on `#0A0F1E` = ~16.5:1 (WCAG AAA). This overrides the light-theme planning reference.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | CSS Var | Value | Usage |
|-------|---------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, inline padding (e.g. tag hash gap) |
| sm | `--space-2` | 8px | Compact element spacing (e.g. tag-row gap) |
| md-sm | `--space-3` | 12px | Card body element gaps, TOC item spacing |
| md | `--space-4` | 16px | Default element spacing, card body padding |
| lg | `--space-5` | 24px | Section padding, card body bottom padding |
| xl | `--space-6` | 32px | Layout gaps |
| 2xl | `--space-7` | 48px | Major section breaks, footer padding |
| 3xl | `--space-8` | 64px | Page-level spacing |
| 4xl | `--space-9` | 96px | Page bottom padding |

### Non-standard-set spacing exceptions

The standard set is {4, 8, 16, 24, 32, 48, 64}. The following values are outside that set but are valid multiples of 4 with explicit justification:

- `--space-3: 12px` — intermediate gap between 8px and 16px needed for card/tag spacing; no standard-set value fills this role without visual crowding (8px) or excess whitespace (16px).
- `--space-9: 96px` — page-bottom breathing room; 64px is too tight for large-viewport layouts; WCAG 2.5.8 touch-target consideration for footer proximity.
- Skip-link `padding: 20px` horizontal — gives adequate touch target width without reaching 24px which would make the chip feel oversized; 20 = 5 × 4 (valid multiple-of-4 extension).

`.site-nav` padding (Phase 1 value, `18px 24px`) is out of scope for this spec — Phase 3 does not modify nav styling.

Additional non-spacing exceptions:
- Focus ring offset: `outline-offset: 3px` (not a spacing token; part of focus specification).

| Destructive | `#FF6BD6` (`--cat-lideranca`) | Used only for `.callout[data-type='danger']` — no destructive actions in Phase 3 |

**Accent (`--nucleo-eletrico`) reserved for:**
1. Focus ring (`:focus-visible` outline)
2. Skip-link visible-state text
3. Prose hyperlinks
4. Drop-cap first letter
5. TOC counter numbers
6. Tag `#` hash prefix (at 0.7 opacity)
7. Tag hover state text
8. Inline `<code>` foreground
9. `.prose h1 em`, `.prose h2 em` emphasis

**NEVER use `--nucleo-eletrico` or `--chama-primaria` as body paragraph text** — `--texto-principal (#FFFFFF)` is the only WCAG-safe body text color on `--abismo-profundo`.

**WCAG compliance summary for Phase 3 new elements:**
- Skip-link focused state: `#00FFFF` text on `#1B293C` bg = ~9.5:1 (AAA)
- Focus ring: `#00FFFF` outline on `#0A0F1E` bg = ~16.5:1 (AAA)
- 404 page body text: `#FFFFFF` on `#0A0F1E` = 21:1 (AAA)
- `/privacidade` body text: `#E6ECF5` on `#0A0F1E` = ~15:1 (AAA)

## Phase 3 Component Specifications

### 1. SEO Component (`src/components/SEO.astro`)

**Nature:** Head-only component — renders inside `<head>`, no visual output.

**Props interface:**
```typescript
interface Props {
  title: string;            // Page-specific title (without site name suffix)
  description: string;      // max 160 characters; caller truncates if needed
  canonicalUrl?: string;    // Overrides auto-computed URL; use for posts with canonical_url frontmatter
  type?: 'website' | 'article';  // default: 'website'
  image?: string;           // Absolute URL to OG image; optional
  pubDate?: Date;           // article only
  updatedDate?: Date;       // article only; falls back to pubDate in JSON-LD
}
```

**Title format:** `{title} · O Sertão será Cloud`
- Site suffix is appended inside SEO component — callers pass only the page-specific title
- Max computed length: 80 chars page title + 22 chars suffix = 102 chars. Callers must keep `title` ≤ 58 chars to stay within 80-char total. Enforce via Zod: `title: z.string().max(80)` (already in schema — sufficient).

**Description truncation:** SEO component truncates `description` to 155 characters with `…` if prop exceeds that length.

**Canonical URL logic:**
1. If `canonicalUrl` prop is provided → use it as-is
2. Otherwise → `new URL(Astro.url.pathname, SITE_URL)` (same pattern as existing BaseLayout)

**Required meta tags emitted:**
```html
<title>{title} · O Sertão será Cloud</title>
<meta name="description" content="{description truncated to 155}" />
<link rel="canonical" href="{canonicalUrl}" />
<meta property="og:type" content="{type}" />
<meta property="og:url" content="{canonicalUrl}" />
<meta property="og:title" content="{title} · O Sertão será Cloud" />
<meta property="og:description" content="{description}" />
<meta property="og:locale" content="pt_BR" />
{image && <meta property="og:image" content="{image}" />}
{type === 'article' && image && <meta property="og:article:published_time" content="{pubDate.toISOString()}" />}
{type === 'article' && updatedDate && <meta property="og:article:modified_time" content="{updatedDate.toISOString()}" />}
<meta name="twitter:card" content="{image ? 'summary_large_image' : 'summary'}" />
<meta name="twitter:site" content="@sertaoseracloud" />
<meta name="twitter:creator" content="@sertaoseracloud" />
{image && <meta name="twitter:image" content="{image}" />}
```

**JSON-LD (article type only):**
```json
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "{title}",
  "description": "{description}",
  "author": {
    "@type": "Person",
    "name": "Cláudio Filipe Lima Rapôso",
    "url": "https://sertaoseracloud.com"
  },
  "publisher": {
    "@type": "Organization",
    "name": "O Sertão será Cloud",
    "url": "https://sertaoseracloud.com"
  },
  "datePublished": "{pubDate.toISOString()}",
  "dateModified": "{(updatedDate ?? pubDate).toISOString()}",
  "image": "{image or omitted if not present}",
  "mainEntityOfPage": {
    "@type": "WebPage",
    "@id": "{canonicalUrl}"
  },
  "inLanguage": "pt-BR"
}
```

**Integration:** BaseLayout replaces its existing inline OG block (lines 35–41 of current `BaseLayout.astro`) with `<SEO ... />`. All existing attributes are preserved — none are removed, only consolidated.

**RSS autodiscovery link (in BaseLayout head, not SEO component):**
```html
<link rel="alternate" type="application/rss+xml" title="O Sertão será Cloud — RSS" href="/rss.xml" />
```
Place this in BaseLayout `<head>` unconditionally — every page announces the feed.

### 2. Skip-Link

**Element:** First focusable element inside `<body>`, before `.ambient` div. In `BaseLayout.astro`.

**HTML:**
```html
<a href="#main-content" class="skip-link">Pular para o conteúdo</a>
```

**Hidden state (default):** Positioned off-screen using transform (not `display:none` — must remain in accessibility tree).

**Visible state (on `:focus-visible`):** Slides in from top-left, fully visible, overlaid above all content.

**CSS specification:**
```css
.skip-link {
  /* Hidden state */
  position: absolute;
  top: 0;
  left: 0;
  transform: translateY(-100%);
  transition: transform 0.15s ease;

  /* Visual design */
  background: var(--sub-nivel);        /* #1B293C */
  color: var(--nucleo-eletrico);       /* #00FFFF — 9.5:1 contrast on #1B293C */
  border: 1px solid var(--hairline-strong);
  padding: 12px 20px;
  font-family: 'Space Grotesk', system-ui, sans-serif;
  font-size: 14px;
  font-weight: 600;
  letter-spacing: 0.02em;
  text-decoration: none;
  z-index: 9999;

  /* Focus state: visible */
  &:focus-visible {
    transform: translateY(0);
    outline: 2px solid var(--nucleo-eletrico);
    outline-offset: 3px;
  }
}
```

**Contrast verification:** `#00FFFF` on `#1B293C` = approximately 9.5:1 (WCAG AAA). Approved.

**Target anchor:** `<main id="main-content">` already exists in `BaseLayout.astro` — no change needed.

### 3. Focus-Visible Ring (Global)

**Applied to:** All interactive elements — `a`, `button`, `input`, `select`, `textarea`, `[tabindex]`, `summary`.

**CSS specification (add to `src/styles/global.css`, section 04 Reset & Base):**
```css
/* Global focus ring — WCAG AA+ compliant */
:focus-visible {
  outline: 2px solid var(--nucleo-eletrico);  /* #00FFFF — 16.5:1 on #0A0F1E */
  outline-offset: 3px;
  /* Do NOT remove outline without this rule — no outline: none anywhere */
}

/* Suppress browser default only where we've replaced it */
:focus:not(:focus-visible) {
  outline: none;
}
```

**Scope:** Applied globally. No component may override with `outline: none` without an equivalent `focus-visible` replacement. This is a hard constraint.

**Exception — nav links:** Existing `.nav-links a` hover styles use `border-color` transitions. The focus-visible ring applies via outline, which renders outside the border box and does not conflict with existing hover border styling.

## Format Date Helper (`src/lib/format-date.ts`)

**Output format:** `"25 de abril de 2026"` (day + "de" + month name in lowercase + year)

**Specification:**
```typescript
export function formatDatePtBr(date: Date): string {
  return new Intl.DateTimeFormat('pt-BR', {
    day: 'numeric',
    month: 'long',
    year: 'numeric',
    timeZone: 'America/Sao_Paulo',
  }).format(date);
}
```
