1. **Import:** `import { SITE_TITLE, SITE_URL, AUTHOR } from '../lib/consts';` (path relative to src/components/)

2. **Props interface:**
```typescript
interface Props {
  title: string;
  description: string;
  canonicalUrl?: string;
  type?: 'website' | 'article';
  image?: string;
  pubDate?: Date;
  updatedDate?: Date;
}
```

3. **Canonical URL logic:** `const canonicalUrl = canonicalUrlProp ?? new URL(Astro.url.pathname, SITE_URL).href;`

4. **Description truncation:** `const desc = description.slice(0, 155) + (description.length > 155 ? '…' : '');`

5. **Title format:** `const fullTitle = \`${title} · ${SITE_TITLE}\`;` — Note: separator is ` · ` (space + middle dot U+00B7 + space), NOT hyphen or pipe.

6. **JSON-LD:** Only render when `type === 'article'`. Use `JSON.stringify()` on the full object and render via Astro's `set:html` on a `<script type="application/ld+json">` tag. JSON-LD shape (all fields):
```json
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": title,
  "description": desc,
  "author": { "@type": "Person", "name": AUTHOR.name, "url": SITE_URL },
  "publisher": { "@type": "Organization", "name": SITE_TITLE, "url": SITE_URL },
  "datePublished": pubDate?.toISOString(),
  "dateModified": (updatedDate ?? pubDate)?.toISOString(),
  // image: only include key if image is truthy
  "mainEntityOfPage": { "@type": "WebPage", "@id": canonicalUrl },
  "inLanguage": "pt-BR"
}
```

7. **Twitter Card:** `summary_large_image` when `image` prop is present, `summary` otherwise. Both `twitter:site` and `twitter:creator` = `@sertaoseracloud`.

8. **OG tags emitted (always):** og:type, og:url, og:title, og:description, og:locale=pt_BR. Conditional: og:image (when image present), og:article:published_time (when type=article AND pubDate), og:article:modified_time (when type=article AND updatedDate).

9. **DO NOT** add `<link rel="alternate" rss+xml>` here — that goes in BaseLayout head directly (Task 2).
10. **DO NOT** add closing `</head>` or any wrapping HTML — this component outputs only head child elements.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| post frontmatter → SEO component | title/description from Markdown frontmatter flows into JSON-LD and OG meta tags |
| post body → rendered HTML | MDX/Markdown content rendered via Astro's render() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | SEO.astro JSON-LD script | mitigate | `JSON.stringify()` escapes all string fields before `set:html` injection — title/description characters like `"`, `<`, `>` are encoded in JSON output |
| T-03-02 | Spoofing | `canonical_url` frontmatter → `<link rel="canonical">` | accept | `canonical_url` used only as `href` value in a `<link>` tag — not a redirect target; no server-side request is made with this value |
| T-03-03 | Tampering | og:image URL from frontmatter | accept | OG image URL is from trusted pipeline (PRBuilder writes it from dev.to cover_image); used only as content attribute, not fetched server-side |
</threat_model>
