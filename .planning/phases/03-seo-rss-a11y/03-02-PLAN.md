  <action>
Create `src/pages/rss.xml.ts` using the official Astro 6 content collection RSS pattern.

CRITICAL DO NOT: Do NOT use `post.compiledContent()` or `post.render()` — these are NOT available on content collection entries in Astro 6. TypeScript will fail with "Property 'compiledContent' does not exist on type 'CollectionEntry'".

USE: `post.body` (raw Markdown string) → `parser.render(post.body)` (markdown-it) → `sanitizeHtml(html, allowlist)` (sanitize-html).

Full implementation:
```typescript
import rss from '@astrojs/rss';
import { getCollection } from 'astro:content';
import sanitizeHtml from 'sanitize-html';
import MarkdownIt from 'markdown-it';
import { SITE_TITLE, SITE_DESCRIPTION } from '../lib/consts';

const parser = new MarkdownIt();

export async function GET(context: { site: URL }) {
  const posts = await getCollection('posts', ({ data }) => !data.draft);
  return rss({
    title: SITE_TITLE,
    description: SITE_DESCRIPTION,
    site: context.site,
    customData: `<language>pt-BR</language>`,
    items: posts
      .sort((a, b) => b.data.pubDate.valueOf() - a.data.pubDate.valueOf())
      .map((post) => ({
        title: post.data.title,
        pubDate: post.data.pubDate,
        description: post.data.description,
        link: `/posts/${post.id.replace(/\.[^.]+$/, '')}/`,
        content: sanitizeHtml(parser.render(post.body ?? ''), {
          allowedTags: sanitizeHtml.defaults.allowedTags.concat(['img']),
        }),
      })),
  });
}
```

Notes on the implementation:
- `const parser = new MarkdownIt()` — instantiated at module scope (not per-request) for performance
- `context.site` — the `site` option in `astro.config.mjs` (`https://sertaoseracloud.com`) is passed as `context.site` at build time
- `customData: \`<language>pt-BR</language>\`` — injects the language element into the channel (per D-07)
- `post.id.replace(/\.[^.]+$/, '')` — strips `.md` extension from collection entry id (same pattern as index.astro card href)
- `post.body ?? ''` — safe fallback for empty posts
- `allowedTags.concat(['img'])` — allows img tags in the sanitized output so cover images appear in RSS readers
- Draft filter: `!data.draft` always (no dev-mode exception — RSS never shows drafts)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| post.body (raw Markdown) → RSS content:encoded | Untrusted Markdown from content collection flows into RSS XML feed consumed by RSS readers |
| robots.txt Sitemap URL | Static URL hardcoded — no user input; only risk is misconfiguration |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | src/pages/rss.xml.ts — markdown-it render → RSS content | mitigate | `sanitize-html` with `sanitizeHtml.defaults.allowedTags.concat(['img'])` cleanses rendered HTML before embedding in feed — removes script tags, event handlers, and XSS vectors |
| T-03-05 | Elevation of Privilege | public/robots.txt — Disallow directives | accept | No `Disallow` lines — entire public site is intentionally crawlable; robots.txt grants no access that isn't already public |
| T-03-06 | Spoofing | RSS feed site URL | accept | `context.site` comes from `astro.config.mjs` `site:` field (hardcoded `https://sertaoseracloud.com`) — no user input; no spoofing vector |
</threat_model>
