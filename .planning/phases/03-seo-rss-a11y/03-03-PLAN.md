**A. Create `src/lib/format-date.ts`:**

```typescript
/**
 * Format a Date as PT-BR long date string in America/Sao_Paulo timezone.
 * Output example: "25 de abril de 2026"
 *
 * TIMEZONE NOTE: Dates created from ISO date strings (e.g. new Date('2026-04-25'))
 * are UTC midnight. America/Sao_Paulo is UTC-3, so this will render as the
 * previous calendar day. Pass dates with an explicit time component (e.g.
 * new Date('2026-04-25T12:00:00')) to stay within the intended day.
 * For frontmatter pubDate values from Zod coerce.date(), this off-by-one is
 * expected behavior — the date represents a calendar day, not a timestamp.
 */
export function formatDatePtBr(date: Date): string {
  return new Intl.DateTimeFormat('pt-BR', {
    day: 'numeric',
    month: 'long',
    year: 'numeric',
    timeZone: 'America/Sao_Paulo',
  }).format(date);
}
```

Single named export, no default export (follows consts.ts convention).

**B. Update `src/styles/global.css` — add focus ring and skip-link CSS:**

Read the file first to identify the section boundary. Look for `/* 04` or `/* Reset` section heading. Add the following CSS block AFTER the Reset & Base section ends and BEFORE any component-specific sections:

```css
/* ============================================================
   05b · Accessibility Primitives
   Focus ring (WCAG AAA) + Skip navigation link
   ============================================================ */

/* Global focus ring — WCAG AAA compliant on dark background
   #00FFFF (--nucleo-eletrico) on #0A0F1E (--abismo-profundo) = 16.5:1
   DO NOT use #284068 here — that color fails WCAG on the dark bg (~1.5:1) */
:focus-visible {
  outline: 2px solid var(--nucleo-eletrico);
  outline-offset: 3px;
}

/* Suppress browser default :focus ring only where :focus-visible replaces it.
   Never use outline: none without this pairing. */
:focus:not(:focus-visible) {
  outline: none;
}

/* Skip navigation link — WCAG 2.1 SC 2.4.1
   Visually hidden until focused (transform, not display:none — stays in a11y tree) */
.skip-link {
  position: absolute;
  top: 0;
  left: 0;
  transform: translateY(-100%);
  transition: transform 0.15s ease;
  background: var(--sub-nivel);
  color: var(--nucleo-eletrico);
  border: 1px solid var(--hairline-strong);
  padding: 12px 20px;
  font-family: 'Space Grotesk', system-ui, sans-serif;
  font-size: 14px;
  font-weight: 600;
  letter-spacing: 0.02em;
  text-decoration: none;
  z-index: 9999;
}

.skip-link:focus-visible {
  transform: translateY(0);
  outline: 2px solid var(--nucleo-eletrico);
  outline-offset: 3px;
}
```

CRITICAL: Use `var(--nucleo-eletrico)` (token reference), NOT the hex `#00FFFF` or `#284068` directly. The early planning docs mentioned `#284068` for focus ring — that was a light-mode palette. The actual design system is dark-first; `#284068` on `#0A0F1E` = ~1.5:1 contrast (WCAG fail).
