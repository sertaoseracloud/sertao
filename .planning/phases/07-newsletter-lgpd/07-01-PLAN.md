```astro
    </p>

    <form
      id="newsletter-form"

      </div>

      <div class="newsletter-consent">

          e consinto com o envio de emails.
        </label>
      </div>
    </form>

    <div
      id="newsletter-success"
      hidden
      role="status"
      tabindex="-1"
      class="newsletter-message newsletter-message--success"
    >
      Obrigado! Confira sua caixa de entrada para confirmar.
    </div>

    <div
      id="newsletter-error"
      hidden
      role="alert"
      tabindex="-1"
      class="newsletter-message newsletter-message--error"
    >
    </div>
  </div>
</section>

<script is:inline>
  (function () {
    var USERNAME = 'REPLACE_WITH_BUTTONDOWN_USERNAME';
    var form = document.getElementById('newsletter-form');
    var successDiv = document.getElementById('newsletter-success');
    var errorDiv = document.getElementById('newsletter-error');
    if (!form) return;
    var submitBtn = form.querySelector('button[type="submit"]');

    form.addEventListener('submit', function (e) {
      e.preventDefault();
      var emailVal = document.getElementById('newsletter-email').value;

      submitBtn.disabled = true;
      submitBtn.textContent = 'Enviando...';
      errorDiv.setAttribute('hidden', '');

      var body = new URLSearchParams();
      body.append('email', emailVal);

      // mode: 'no-cors' — response is opaque; cannot read status code.
      // A resolved fetch (no throw) means the request reached Buttondown.
      // Network failures (DNS, timeout, Buttondown down) go to catch.
      fetch(
        'https://buttondown.com/api/emails/embed-subscribe/' + USERNAME,
        { method: 'POST', body: body, mode: 'no-cors' }
      )
        .then(function () {
          // Opaque response — fetch resolved = request reached server
          form.style.display = 'none';
          successDiv.removeAttribute('hidden');
          successDiv.focus();
        })
        .catch(function () {
          // Network failure only (no-cors cannot distinguish 200 from 400)
          submitBtn.disabled = false;
          submitBtn.textContent = 'Inscrever-se';
          errorDiv.textContent =
            'Algo deu errado. Tente novamente ou entre em contato: engcfraposo@gmail.com';
          errorDiv.removeAttribute('hidden');
          errorDiv.focus();
        });
    });
  })();
</script>
```

- The `is:inline` script intercepts submit via `e.preventDefault()` and fires the AJAX path.
- Do NOT add `client:` directive — this is a plain Astro component, not a framework component.
- `sr-only` class is already defined in global.css from Phase 3 (confirmed existing).
- Do NOT add `checked` attribute to the consent checkbox (D-03).
- The `role="status"` on successDiv and `role="alert"` on errorDiv make them ARIA live regions — screen readers announce content changes automatically.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Buttondown embed endpoint | User-supplied email crosses this boundary; URLSearchParams encoding prevents injection |
| User → LGPD consent checkbox | Browser `required` enforces consent gate before submission |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | newsletter-form submit handler | mitigate | Use `new URLSearchParams()` (not `FormData`) to send as `application/x-www-form-urlencoded`; prevents multipart encoding issues; no innerHTML with user-supplied content |
| T-07-02 | Tampering | consent checkbox | mitigate | `required` attribute on checkbox; browser native validation blocks submit when unchecked; no pre-checked attribute (D-03 locked) |
| T-07-03 | Denial of Service | Buttondown embed endpoint | accept | Buttondown rate-limits at service layer; double opt-in prevents activated spam subscriptions |
| T-07-04 | Information Disclosure | is:inline USERNAME placeholder | accept | Username is not a secret — it is the public Buttondown newsletter slug, visible in any subscriber confirmation email |
| T-07-05 | Tampering | XSS via email input reflected in DOM | mitigate | Email value is sent to Buttondown API only (never injected into the DOM via innerHTML); `errorDiv.textContent = ...` (not innerHTML) for error copy |
</threat_model>
