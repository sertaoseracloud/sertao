## Spacing Scale

Declared values — mirrors existing `--space-*` tokens in `global.css`:

| Token | CSS Var | Value | Usage |
|-------|---------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, hash prefix gap in tags |
| sm | `--space-2` | 8px | Tag row gap, inline tight spacing, form field row gap |
| md | `--space-4` | 16px | Default element padding, input padding, consent row margin-top |
| lg | `--space-5` | 24px | Section padding, form group gap, button horizontal padding |
| xl | `--space-6` | 32px | Post header margin-bottom |
| 2xl | `--space-7` | 48px | Major section breaks (e.g., comments-section margin-top) |
| 3xl | `--space-8` | 64px | Clear break between article prose and newsletter section |

Exceptions:
- Newsletter section top-margin: `var(--space-8)` (64px) — mirrors `comments-section` margin-top pattern, creates equal visual weight between the two post-article sections.
- LGPD checkbox touch target minimum: 44px height — accessibility requirement (WCAG 2.5.3). Achieved via `min-height: 44px` on the label wrapper, not padding change.
- Email input height: 44px — matches touch target recommendation for form inputs.

**Source:** `src/styles/global.css` `--space-*` custom properties; `comments-section` pattern from Phase 6.

## Color

| Role | Token | Dark Value | Light Value | Usage |
|------|-------|-----------|-------------|-------|
| Dominant (60%) | `--abismo-profundo` | `#0A0F1E` | `#F5F0E8` | Page background, ambient layer |
| Secondary (30%) | `--sub-nivel` | `#1B293C` | `#E8E3D8` | Newsletter section background, email input background, card surfaces |
| Accent (10%) | `--nucleo-eletrico` | `#00FFFF` | `#284068` | CTA button background, focus ring, section heading color accent, link color |
| Destructive | n/a | n/a | n/a | No destructive actions in this phase |
| State tints | n/a | `rgba(107,255,180,0.4)` / `rgba(255,107,214,0.35)` | same | Success border/bg tint; error border/bg tint — non-accent status indicators only, never used for text |

**Accent reserved for:**
- Subscribe CTA button background: `var(--nucleo-eletrico)` with text color `var(--abismo-profundo)` (ensures WCAG AA contrast)
- Focus ring on all form inputs and the LGPD checkbox: `2px solid var(--nucleo-eletrico)` via existing `:focus-visible` rule
- Links within LGPD checkbox label (`/privacidade`) and within prose text
- The `#` hash prefix in tag chips (existing pattern; not introduced by this phase)

**CTA button contrast check:**
- Dark mode: `#00FFFF` on `#1A3AC8` (button bg from `--chama-primaria`) — The CTA button uses `background: var(--nucleo-eletrico)` with `color: var(--abismo-profundo)` (`#0A0F1E`). Ratio: #00FFFF on #0A0F1E = 16.5:1 (WCAG AAA). Use this pattern.
- Light mode: `--nucleo-eletrico` overrides to `#284068`; CTA button keeps `background: var(--chama-primaria)` (`#1A3AC8`) with `color: var(--texto-principal)` (`#0A0F1E`). Verified safe.

**State tints** (`rgba(107,255,180,0.4)` for success and `rgba(255,107,214,0.35)` for error) are used exclusively for border and background of the inline feedback messages. These are non-accent, non-semantic-text colors — they are not applied to any text or interactive elements.

**Source:** `src/styles/global.css` design tokens; `CLAUDE.md` WCAG notes; `global.css` `[data-theme="light"]` overrides.

## Interaction Contract

### Form Submit Flow (AJAX, `<script is:inline>`)

1. User clicks "Inscrever-se" — submit button immediately disabled, label changes to "Enviando..."
2. `fetch('https://buttondown.com/api/emails/embed-subscribe/{username}', { method: 'POST', body: formData })` — `Content-Type: application/x-www-form-urlencoded`
3. **HTTP 200:** Hide form wrapper (`display:none`), show `#newsletter-success` div with success copy.
4. **HTTP 400 (already subscribed):** Hide form wrapper, show `#newsletter-error` with already-subscribed copy.
5. **Network error / non-200:** Re-enable submit button, show `#newsletter-error` with generic error copy. Form remains visible so user can retry.

The LGPD checkbox is `required` — browser native validation blocks submit if unchecked. No custom JS validation needed for checkbox.

### Focus Management

- On success/error: `document.getElementById('newsletter-success').focus()` (or error div) — the revealed div must have `tabindex="-1"` so `.focus()` works. This ensures screen reader users hear the outcome without having to navigate.
- Form inputs follow existing `:focus-visible` rule in `global.css` — no additional CSS needed.

## Accessibility Requirements

- `<input type="email">` must have `required` and an associated `<label>` (even if visually hidden via `sr-only` style or `aria-label`).
- `<input type="checkbox">` must have an associated `<label for="...">` — never `aria-label` alone for checkboxes.
- LGPD checkbox: `required` attribute enforces browser native validation. Error message from browser default is acceptable; no custom validation UI needed.
- Success/error message divs: `tabindex="-1"` + `.focus()` call on state change for screen reader announcement.
- Success/error divs: `role="status"` (success) and `role="alert"` (error) — ensures ARIA live region announcement without needing to call `.focus()` as a fallback.
- Newsletter section `<h2>` must use `aria-labelledby` on the `<section>` — matches `comments-section` pattern.
- `/newsletter` page: `<h1>` must be present and unique. The site title suffix in `<title>` element confirms page identity.
